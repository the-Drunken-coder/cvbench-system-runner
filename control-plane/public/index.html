What CVBench measures

A leaderboard with receipts.

Every score is backed by deterministic matching, external timestamps, resource samples, and evidence artifacts. One flattering average cannot hide system failure.

A

Accuracy

Detection, localization, classification, and confidence calibration against frame-level ground truth.

R

Robustness

Acquisition, visible retention, identity continuity, occlusion recovery, and fault response.

L

Latency

Receive time minus source-frame time, recorded by the harness—not self-reported by the model.

Ω

Resources

CPU, memory, process count, and optional GPU telemetry sampled outside the submitted container.

!

Diagnostics

Structured findings and portable evidence packets for the largest observed failures.

02

The exact container contract

Bring one immutable Linux image.

CVBench runs submitted code only on ephemeral GitHub-hosted Linux runners. Cloudflare stores and serves control-plane data; it never executes your model.

1

Ship a digest

Use a public or runner-accessible linux/amd64 OCI image pinned as registry/repo@sha256:…. Tags are rejected.

2

Connect and become ready

Read CVBENCH_INPUT_SOCKET, connect to /run/cvbench/input.sock, then print exactly CVBENCH_READY as one stdout line.

3

Consume ordered frames

The same Unix stream stays open for the sequence. Keep temporal or multi-frame state in memory. Messages contain lengths, JSON metadata, then JPEG bytes.

4

Emit tracks

Print one cvbench.track/v1 JSON object per stdout line. Put logs on stderr. The harness timestamps receipt externally.

03

Submit once, retry safely

One API for humans and agents.

A submission key creates jobs. Reads are public. Every create request needs a stable idempotency key, so retries cannot accidentally enqueue duplicates.

POST /api/v1/submissions
curl -sS "$CVBENCH_API_BASE_URL/api/v1/submissions" \
  -H "Authorization: Bearer $CVBENCH_API_KEY" \
  -H "Idempotency-Key: my-model-v7-001" \
  -H "Content-Type: application/json" \
  --data '{
    "image": "ghcr.io/acme/tracker@sha256:0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef",
    "argv": ["python", "-m", "tracker"],
    "name": "Acme Temporal Tracker",
    "model_version": "7.0"
  }'
import os, uuid, requests

response = requests.post(
    f"{os.environ['CVBENCH_API_BASE_URL']}/api/v1/submissions",
    headers={
        "Authorization": f"Bearer {os.environ['CVBENCH_API_KEY']}",
        "Idempotency-Key": str(uuid.uuid4()),
    },
    json={
        "image": "ghcr.io/acme/tracker@sha256:...",
        "argv": ["python", "-m", "tracker"],
        "name": "Acme Temporal Tracker",
        "model_version": "7.0",
    },
    timeout=30,
)
response.raise_for_status()
print(response.json()["id"])
Goal: win the public CVBench benchmark.

1. GET /api/v1/contract and /api/v1/openapi.json.
2. Build and publish a Linux/amd64 OCI image.
3. Resolve its registry digest. Never submit a mutable tag.
4. POST image + argv + metadata with bearer auth and a unique
   Idempotency-Key. Do not send source, build steps, shell, or env.
5. Poll the returned Location until status is succeeded or failed.
6. Inspect result.metrics and diagnostics, improve the system,
   publish a new digest, and submit with a new idempotency key.
Need a key? The operator bootstraps one or more keys into the Worker secret SUBMISSION_API_KEYS. Keys are comma-separated for simple rotation and are never stored in job records.

05

Security boundary

The queue is not the sandbox.

The control plane validates intent. The existing CVBench engine enforces execution isolation on a disposable runner.

01Cloudflare WorkerStatic assets, API validation, authentication, rate limits, and D1 state only.

02GitHub-hosted runnerLeases one job, pulls the exact digest, runs CVBench, and posts the report back.

03Tested containerNo network, no Docker socket, an unprivileged UID/GID, CPU and memory caps, and only the progressive socket mount.

04Secret separationRunner and callback tokens stay in the host process environment and are removed from the benchmark subprocess environment.
